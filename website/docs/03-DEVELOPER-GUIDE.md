# inaffi — Developer Guide

> Practical guide for maintaining and extending the PHP codebase.
> The WT **`v1` branch** is the reference for all design, layout, and feature decisions.
> Read the relevant WT `v1` source file BEFORE building or modifying any component.

---

## First Thing Every Session

```bash
# 1. Fetch latest WT v1 reference
git fetch wt v1

# 2. Read the relevant WT source before touching anything
git show wt/v1:website/app/globals.css               # colors
git show wt/v1:website/app/page.tsx                  # all 6 homepage sections
git show wt/v1:website/components/layout/Header.tsx  # header
git show wt/v1:website/components/layout/Footer.tsx  # footer
git show wt/v1:website/lib/config.ts                 # platforms, categories, constraints
git show wt/v1:website/app/\[username\]/page.tsx     # storefront
git show wt/v1:website/app/dashboard/page.tsx        # dashboard

# 3. Check current state
git log --oneline -5
```

## PHP Coding Patterns

### Database — always PDO prepared statements

```php
// Single row
$stmt = get_db()->prepare('SELECT * FROM creators WHERE username = ? LIMIT 1');
$stmt->execute([$username]);
$creator = $stmt->fetch();  // assoc array or false

// Multiple rows
$stmt = get_db()->prepare('SELECT * FROM outfits WHERE creator_id = ? ORDER BY created_at DESC');
$stmt->execute([$id]);
$outfits = $stmt->fetchAll();

// Scalar / count
$stmt = get_db()->prepare('SELECT COUNT(*) FROM clicks WHERE creator_id = ?');
$stmt->execute([$id]);
$total = (int) $stmt->fetchColumn();
```

### Output — ALWAYS use `e()`

```php
echo e($user['display_name']);     // ✅ safe — htmlspecialchars()
echo $user['display_name'];        // ❌ XSS risk
```

### URLs

```php
site_url('dashboard')              // → https://inaffi.com/dashboard
site_url('uploads/outfits/x.jpg')  // → https://inaffi.com/uploads/outfits/x.jpg
```

### File paths — `dirname(__DIR__)`, NOT `$_SERVER['DOCUMENT_ROOT']`

```php
// ✅ Works at any Hostinger deploy path
$app_root  = dirname(__DIR__);                    // from includes/ → inaffi/
$full_path = $app_root . '/uploads/outfits/' . $filename;

// ❌ Breaks on Hostinger subdirectory deploy
$full_path = $_SERVER['DOCUMENT_ROOT'] . '/uploads/' . $filename;
```

### Auth guards

```php
require_login();              // redirects to /login if no session
require_admin();              // require_login() + check is_admin = 1
$creator = get_current_creator();  // returns full creator row array
```

### CSRF — on every POST form

```php
// In GET handler (render form):
$csrf = generate_csrf_token();
// In form HTML:
// <input type="hidden" name="csrf_token" value="<?= e($csrf) ?>">
// In POST handler (before any DB write):
verify_csrf_token($_POST['csrf_token'] ?? '');
```

### Flash messages

```php
set_flash('Saved successfully!', 'success');  // 'success' | 'error' | 'info'
redirect(site_url('dashboard'));
// render_flash() in header.php outputs the message on next load
```

### Image uploads

```php
// In POST handler:
$path = save_image($_FILES['outfit_image'], 'outfits');
// Returns "uploads/outfits/a3f8c2d1.jpg" (store in DB) or null if no file

// On delete:
delete_image($outfit['image']);  // pass relative path from DB
```

## Common Tasks

### Rename the Platform
1. Edit `includes/config.php` → change `SITE_NAME` and `SITE_TAGLINE`

### Change Colors
1. `git show wt/v1:website/app/globals.css` — read exact values
2. Edit `assets/css/styles.css` → update `:root` CSS variables

### Add a New Platform
1. Add to `$PLATFORMS` in `config.php`
2. Add badge colors to `$PLATFORM_COLORS`
3. Add logo path to `$PLATFORM_LOGOS`
4. Upload 64×64px PNG to `assets/images/platforms/` on server

### Add a New Category
1. Add to `$CATEGORIES` in `config.php` — dropdowns and filters auto-update

### Add a New Dashboard Page
1. Create `dashboard/your-page.php` with `require_login()` at top
2. Add nav link to `components/dashboard-sidebar.php`
3. Add `.htaccess` rule: `RewriteRule ^dashboard/your-page/?$ dashboard/your-page.php [L,QSA]`

### Add a New Public Page
1. Create `your-page.php` with header + footer includes
2. Add `.htaccess` rule before the username catch-all

### Make Account Admin
```sql
-- In Hostinger phpMyAdmin → SQL tab:
UPDATE creators SET is_admin = 1 WHERE email = 'Admin@inaffi.com';
```

## Troubleshooting

| Problem | Cause | Fix |
|---|---|---|
| "Database connection failed" | Wrong credentials in `config.php` | Check DB_USER/DB_PASS in hPanel → Databases |
| DB connects but queries fail | Old username `u789815590_admin` in config | Update to `u789815590_inaffi_user` |
| Images not showing | `uploads/` missing or wrong permissions | Create folders, `chmod 755` |
| Routes returning 404 | mod_rewrite not active | hPanel → Apache modules → enable mod_rewrite |
| CSS not updating | Browser cache | Hard refresh `Ctrl+Shift+R` or append `?v=N` to CSS link |
| CSRF error on POST | Session expired | Re-login and try again |
| Platform logo missing | PNG not on server | Upload to `assets/images/platforms/` |
| Homepage outfit card empty | No featured outfit | Dashboard → any outfit → ⭐ Feature |
| Image upload fails | GD missing or wrong path | Check PHP extensions; verify `uploads/` chmod 755 |
| Admin "Access denied" | `is_admin` not set | `UPDATE creators SET is_admin=1 WHERE email=?` |
| `dirname(__DIR__)` wrong path | Wrong file depth | Check which directory the include is in |
| Git push SSL error | Corporate proxy | `git -c http.sslVerify=false push github main` |
| Hostinger shows old code | GitHub not updated first | Push GitHub before triggering Hostinger deploy |
| Gold color looks wrong/bright | Using `#D6B25E` (master branch) | Read from `wt/v1:website/app/globals.css` → `#C9A96E` |
| Section bg not alternating | Wrong token name | Use `--color-section-alt` for alt sections |
| Buttons not pill-shaped | Missing `border-radius: 9999px` | Use `var(--btn-radius)` |
| Footer looks like 2-col | Old footer not updated | Build 4-col footer matching WT v1 `Footer.tsx` |
