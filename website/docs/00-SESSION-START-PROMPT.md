## PROJECT QUICK-START

### Stack

| Layer | Technology |
|---|---|
| Server | PHP 8.x on Hostinger shared hosting |
| Database | MySQL (Hostinger) |
| Auth | PHP Sessions + bcrypt |
| Images | Local `uploads/` folder + GD compression |
| Frontend | Vanilla HTML/CSS/JS — no build step |
| Deploy | GitHub (`bmoglade/INAFFI_Website`) → Hostinger auto-pull |

| File | Purpose |
|---|---|
| `website/inaffi/includes/config.php` | Site config — DB, platforms, categories (NOT in git) |
| `website/inaffi/includes/config.example.php` | Committed template for config |
| `website/inaffi/assets/css/styles.css` | Full design system (→ WT `globals.css`) |
| `website/inaffi/index.php` | Homepage (→ WT `app/page.tsx`) |
| `website/inaffi/storefront.php` | Creator storefront (→ WT `app/[username]/page.tsx`) |
| `website/inaffi/components/` | Shared PHP partials (→ WT `components/`) |
| `website/inaffi/dashboard/` | Creator dashboard pages |
| `website/inaffi/admin/index.php` | Admin panel |
| `website/docs/schema.sql` | Full MySQL schema |

### Deploy Workflow

```bash
git add . && git commit -m "description"
git push origin master                               # → Foundry (WT_2 repo)
git -c http.sslVerify=false push github main         # → GitHub (Hostinger source)
# Then: hPanel → Git → Deploy/Pull
# Test: https://inaffi.com
```

## SESSION RULES

1. **Read WT v1 source FIRST** before building or modifying any UI component
2. **Use exact token values** from WT v1 `globals.css` — never invent colors
3. **All buttons are pills** (`border-radius: 9999px`) — no square buttons
4. **Always use `e()` for output** — `echo e($var)` not `echo $var`
5. **Always use `dirname(__DIR__)`** not `$_SERVER['DOCUMENT_ROOT']` for file paths
6. **PDO prepared statements only** — never string interpolation in SQL
7. **Update docs + CHANGELOG in the same commit** as the code change
